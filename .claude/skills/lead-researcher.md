# Skill: 目标客户研究 (Lead Researcher)

## 触发条件
当用户提到以下意图时激活：
- "找客户"、"找目标"、"找潜在客户"、"筛选客户"
- "谁适合我的产品"、"帮我找一批品牌"
- "这个列表里哪些值得联系"
- outreach-campaign 流程中的第 1 步

## 概述
帮用户找到并筛选值得 outreach 的目标客户。**不限输入格式、不限行业**——用户可以给一份表格、一组 URL、一段文字描述，甚至只说"帮我找 XX 类客户"。

本 skill 的核心能力是：理解用户的产品 → 判断谁是合适的客户 → 输出一份排好优先级的目标列表。

## 被其他 Skill 调用
- **outreach-campaign**（`.claude/skills/outreach-campaign.md`）在第 1 步调用本 skill
- 本 skill 也可被用户直接触发，独立使用

## 前置输入
开始前必须已知：
1. **用户的产品/服务描述** — 解决什么问题、目标客户是谁、核心卖点
2. **客户来源** — 以下任一形式均可（见"输入处理"）

## 输入处理

### 支持的输入格式

| 输入形式 | 处理方式 |
|---------|---------|
| Excel/CSV 文件 | 用 `input_parser.py` 读取，Claude 自行理解每列含义 |
| URL 列表 | 逐个 WebFetch/浏览器访问，提取品牌信息 |
| 品牌名列表 | WebSearch 搜索每个品牌，补充信息 |
| 文字描述 | "帮我找 50 个做 DTC 护肤的品牌" → Claude 自主搜索 |
| 飞书/Google Doc | 用 WebFetch 或浏览器读取文档内容 |

### Excel/CSV 文件处理

用 `scripts/outreach/input_parser.py` 读取文件：

```bash
python3 scripts/outreach/input_parser.py "<文件路径>"
python3 scripts/outreach/input_parser.py "<文件路径>" --summary
```

解析器会自动读取所有列，不预设任何字段名。Claude 根据列名和数据内容自行判断哪些信息对筛选有用。

**输出给用户**：
- 解析到多少条记录
- 识别到哪些字段
- 展示前 3-5 条示例让用户确认

**如果解析失败**：
- 文件不存在 → 提示检查路径
- 格式不支持 → 说明支持 .csv 和 .xlsx
- 缺少 openpyxl → 提示 `pip3 install openpyxl`

### 无文件输入处理

如果用户没有现成列表，Claude 主动帮忙构建：

1. 根据用户的产品描述，确认理想客户画像（行业、规模、地区等）
2. 用 WebSearch 搜索匹配的品牌/公司
3. 对每个找到的目标，提取关键信息（品牌名、官网、品类、规模等）
4. 整理成列表进入筛选环节

## AI 筛选与优先级排序

### 筛选原则

**不使用固定字段评分**——Claude 根据用户的产品描述，自行判断哪些维度与匹配度相关。

例如：
- 如果用户卖 Shopify 插件 → 看对方是否用 Shopify、店铺规模、品类匹配度
- 如果用户卖 B2B SaaS → 看公司规模、行业、现有工具栈
- 如果用户做内容营销服务 → 看社媒活跃度、内容质量、粉丝量

### 筛选流程

1. **准备评分数据**：用 `lead_scorer.py` 的 `prepare_scoring_batch()` 将所有可用字段打包（不挑选特定字段，透传全部信息）
2. **构建评分 prompt**：用 `build_scoring_prompt()` 生成，包含用户产品描述和客户信息
3. **Claude 评分**：在对话中直接完成（不需要外部 API），输出三档分组
4. **解析结果**：用 `parse_scoring_result()` 提取 JSON
5. **合并回列表**：用 `merge_scores()` 写入评分

### 评分输出

三档分组：
- 🟢 **高优**：高度匹配，值得优先联系
- 🟡 **中优**：有一定匹配度，可以联系
- 🔴 **不推荐**：匹配度低，建议跳过

每个评分附带一句话理由。

### 交互节点

```
以上是筛选结果。你可以：
1. 确认分组，继续下一步
2. 调整某些客户的优先级
3. 补充产品描述信息，重新筛选

请问怎么处理？
```

### 注意事项
- 超过 20 个客户分批处理（每批 20 个）
- 评分在 Claude 对话中完成，不需要额外 API 调用
- 如果客户信息太少（只有品牌名），先用 WebSearch 补充信息再评分

## 输出格式

无论输入来源是什么，最终输出统一格式：

```json
[
  {
    "brand_name": "品牌名",
    "website": "https://...",
    "email": "xxx@xxx.com",  // 可能为空
    "priority": "high",
    "score_reason": "一句话理由",
    // ... 其他从数据源获取的字段（全部保留）
  }
]
```

保存到 `/tmp/outreach_merchants_scored.json`，供后续 skill 使用。
